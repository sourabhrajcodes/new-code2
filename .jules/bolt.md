## 2026-06-29 - Zustand Selectors and React.memo Optimization
**Learning:** Destructuring the entire state object from a Zustand store (e.g., `const { a, b } = useStore()` instead of `const a = useStore(state => state.a)`) causes the component to subscribe to *all* state changes, leading to unnecessary re-renders when unrelated state changes. Additionally, rendering many elements on a canvas (like Konva) without `React.memo` on the wrapper components causes all elements to re-render when just one element's state changes (like when it's being dragged).
**Action:** Always use granular selectors when using Zustand to prevent unnecessary subscriptions. Wrap complex individual items in a list (like canvas elements) with `React.memo` (and a custom comparison function if necessary) to prevent O(N) re-renders when only a single element is changing.
